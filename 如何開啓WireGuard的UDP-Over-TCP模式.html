如何開啓WireGuard的UDP Over TCP模式

最近，我一直在使用 WireGuard 來加速遊戲，白天的效果令我滿意，但是晚上就不能夠愉快使用了，經歷了各種卡噸和掉線。看下客戶端日誌，連接斷了卡在了握手驗證密鑰過程，我想是網路運營商的 UDP 干擾造成了這一問題。如果要用 TCP 傳輸數據，那我就能用了吧。相比其他工具， WireGuard 第三方客戶端 TunSafe 可以簡單實現 UDP Over TCP 這一效果，只需要更改一下 Server 和 Client 的配置文件即可。

Server配置

安裝 TunSafe

設置tun0

由於 TunSafe 的 Interface 是 tun0, 我們需要給予它連接互聯網的權限,另外 ens3 要改成你 Server 的 Internet Interface, 如果你是第一次安裝請參考一下我之前寫的內容。

啓動 TunSafe

因爲我的 Server 上已配置過 WireGuard, 所以我就偷懶直接改已有配置文件 wg0
將 [Interface] 中的 ListenPort 改爲 ListenPortTCP 保存後將 TunSafe 啓動,如果沒有報錯，就說明啓動成功了

Client配置

下載 1.5-rc2 及以上版本客戶端， 將 Endpoint = ip:51820 改爲 Endpoint=tcp://5.5.5.5:1234

總結

我在 Windows 10 測試了一下遊戲，效果非常不理想，對它我已經解毒了。雖然丟包減少了，但原來300+延遲現在變成最高600+，而且TunSafe 1.5-rc2 客戶端不穩定有時候會掉線，從而導致遊戲掉線。如果服務器延遲本身就很高的話，建議就不要嘗試了。
